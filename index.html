类的加载与反射

类的加载,连接,初始化

当系统主动使用某个类时,如果该类还未被加载到内存中,则系统会通过加载、连接、初始化三个步骤对该类初始化。

  • 类的加载

    类的加载由类加载器完成，类加载器通常由JVM提供，JVMT提供的这些类加载器同常被称为系统加载器。开发者可以通过继承ClassLoader基类来创建自己的类加载器。
    通过使用不同的类加载器，可以从不同来源加载类的二进制数据，通常有如下几种来源。

    • 本地文件系统加载class文件。
    • 从JAR包加载class文件
    • 通过网络加载class文件
    • 把一个JAVA源文件动态编译，并进行加载。

  • 类的连接

    当类被加载之后，系统同为之生成一个对应的Class对象，接着将会进入连接阶段，连接阶段负责把类的二进制数据合并到JRE中。连接可分为如下三个阶段

    1. 验证：验证阶段用于检验被加载的类是否由正确的内部结构，并和其他类协调一致。
    2. 准备：类的准备阶段则负责为类的类变量分配内存，并设置默认初始值。
    3. 解析：将类的二进制数据中的符号引用替换成直接引用。

  • 类的初始化

    类的初始化主要为声明变量指定初始值，然后使用静态初始化块指定初始值。

    1. 加入这个类还没有被加载和连接，则程序先加载并连接该类。
    2. 假如该类的直接父类还没有被初始化，则先初始化其直接父类。 父类的初始化也是从1～3
    3. 假如类中由初始化语句，则系统按照从上到下的顺序依次执行初始化语句。

类的初始化时机
要么主动创建，要么系统创建。当执行某个操作需要访问类的地址空间，系统就会创建这个类。注意的是final型的类变量，如果值在编译时就定下来，使用到静态类变量，不会导致该类的初始化。如果编译时不能确定，就会导致初始化。
使用ClassLoader类的loadClass()方法来夹杂某个类时，该方法只是加载该类，不会初始化。但是使用Class的forName()就会导致初始化。

    * 通过反射来创建实例

    1. 调用某个类的类方法(静态方法)。
    2. 访问某个类或接口的类变量，或为该类变量赋值。
    3. 使用反射方式来强制创建某个类或接口对应的java.lang.Class对象。
    4. 初始化某个类的子类。

MySQL入门

MySQL是关系型数据。还有一些网状型，层次型，面向对象数据库。

SQL语句基础

SQL全称是(Structured Query Language,结构化查询语言)
标准SQL语句通常分为如下几种类型：

  • 查询语句：主要由select关键字完成。
  • DML(Data Manipulation Language,数据操作语句):主要由insert,update和delete三个关键字完成。
  • DDL(Data Definition Language,数据定义语句):主要由create,alter,drop和truncate四个关键字完成。
  • DCL(Data Control Language,数据控制语句):主要由grant和revoke两个关键字完成。
  • 事物控制语句：主要由commit，rollback，savepoint三个关键字完成。

DDL语句

DDL是操作数据对象的语句，包括创建(create),删除(drop)，修改(alter)数据库对象。下面是常见的数据对象：

对象 对应保留字 描述
表 table 表是存储数据的逻辑单元，列就是字段，行就是记录
数据字典 系统表，用于存放数据相关信息的表。一般由数据库系统维护。
约束 constraint 执行数据校验的规则，用于保证数据完整性的规则。
视图 view 一个或多个数据表里数据的逻辑显示。视图不存储数据。
索引 index 用于提高查询性能，相当于书的目录
函数 function 用于完成一次特定的计算，具有一个返回值
存储过程 procedure 用于王超一次完整的业务处理，没有返回值，但可以通过传出参数将多个返回值传递给调用环境
触发器 trigger 相当于一个事件监听器，当数据发生特定事件后，触发器被触发，完成相应的处理

数据库约束

  • NOTNULL:非空约束，指定某个列不能为空。
  • UNIQUE:唯一约束，指定某列或者几列组合不能重复。
    • 虽然唯一约束的列不可以出现重复值，但可以出现多个null
    • 当为某列创建唯一约束时，MySQL会为该列相应地创建唯一索引，如果不给唯一索引起名，则默认于列名相同。
    • 可以由多列组成唯一约束。
  • PRIMARY KEY:主键，指定该列的值可以唯一标示该记录。
    • 每个表最多只有一个主键，该列不允许出现null。
    • 创建主键，自动创建对应的唯一索引。
    • 可以由多列组成唯一主键。
  • FOREIGN KEY:外键，指定该行从属于主表中的一条记录，主要用于保证参照完整性。
    • 当创建外键时，自动创建索引。
  • CHECK：检查指定一个布尔表达式，用于指定对应的列必须满足该表达式。（MySQL中不会起作用)

索引

索引是存放在模式(schema)中的一个数据库对象。
好处是可以加速查询，但索引也有如下两个坏处：

  • 当记录被添加，删除，修改是，数据库系统要维护索引，有一定的系统开销。
  • 存储索引需要额外的磁盘空间。

视图

视图的作用：

  • 可以限制对数据的访问。
  • 可以使复杂的查询变得简单。
  • 提供了数据的独立性。
  • 提供了对相同数据的不同显示。

对象的共享

  • 重排序

    在没有同步的情况下，编译器、处理器以及运行时等都可能对操作的执行顺序进行一些意想不到的调整。在缺乏足够同步的多线程程序中，要想对内存造作的执行顺序进行判断，几乎无法得出正确的结论。

  • 非原子的64位操作
    在JVM中，允许讲64位的读操作或写操作分解位两个32操作。当读取一个非volatile类型的long变量时，如果对该变量的读操作和写操作在不同的线程中执行，那么很可能会读取到某个值的高32位和另一个值的低32位。
    在多线程中使用共享且可变的long和double等类型变量是不安全的。应该用volatile来声明，或者用锁保护起来。

线程安全性

  • 无状态对象一定是线程安全的
    无状态对象：不包含任何和域，也不包含任何对其他类中域的引用。计算过程中的临时状态仅存在域线程栈上的局部变量中，并且只能由正在执行的线程访问。

  • 竞态条件
    当某个计算的正确性取决域多个线程交替执行的时序是，那么就会发生竞态条件。
    一般竞态条件的本质就是——基于一种可能失效的观察结果来做出判断或者执行某个计算，这种类型的竞态条件称为“先检查后执行”：首先观察到某个条件状态为A，然后根据这个观察结果采取相应的动作，但事实上，在采取动作之前，该状态已经不是A了，从而导致了不可预测的结果。

  • 同步代码块 Synchronized Block
    同步代码块分为两个部分，一个作为锁对象的引用，一个作为由这个锁保护的代码块。是这个代码块变成一个原子操作。

因为没次只有一个线程能执行内置锁保护的代码块，所以这个代码块其实是被原子化了，变成了一个原子操作。虽然保证了线程安全，但是要注意并发性问题，做不好就会降低并发性。

  • 重入
    一个线程试图获取一个已经由他自己持有的锁，这个请求会成功，就是重入。
